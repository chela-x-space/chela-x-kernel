---

# 1. Purpose

This specification defines the canonical classification of Enterprise
Events within the CHELA-X Kernel.

Every Event shall belong to exactly one primary Event Category.

Event Categories provide semantic organization without changing Event
Envelope semantics.

---

# 2. Objectives

The Event Category model provides:

- Enterprise consistency
- Predictable routing
- Stable naming
- Audit organization
- Event discovery
- Traceability
- Future stream partitioning

Categories classify Events.

Categories do not define Event Payload.

---

# 3. Scope

This specification defines:

- Primary Event Categories
- Naming rules
- Category ownership
- Category boundaries

This specification does not define:

- Event Streams
- Replay
- Transport
- Persistence
- Subscribers
- Message Brokers

These capabilities belong to later K5 milestones.

---

# 4. Design Principles

Every Event belongs to exactly one primary category.

Categories are:

- Stable
- Deterministic
- Enterprise-wide
- Immutable after publication
- Independent of storage
- Independent of transport

Categories classify meaning.

They do not classify infrastructure.

---

# 5. Category Hierarchy

Enterprise Event

    │

    ├── System

    ├── Runtime

    ├── Workflow

    ├── Task

    ├── Execution

    ├── Memory

    ├── Security

    ├── Audit

    ├── API

    └── Studio

Every Event Type belongs to one of these categories.

---

# 6. Naming Convention

Canonical Event Types follow:

category.subject.action

Examples:

runtime.health.assessed

workflow.started

task.completed

execution.failed

memory.record.created

security.authorization.denied

audit.entry.recorded

api.request.accepted

studio.command.executed

system.started

Category names remain lowercase.

Words are separated using periods.

Naming shall remain deterministic.

---

# 7. Category Ownership

Each category owns only its own semantic space.

Runtime Events describe Runtime facts.

Workflow Events describe Workflow facts.

Task Events describe Task facts.

Categories shall never overlap in meaning.

---

# 8. Canonical Determination

This specification establishes the canonical Event Category model for
the CHELA-X Enterprise Event System.

Subsequent K5 specifications shall extend these categories without
changing their fundamental semantics.
